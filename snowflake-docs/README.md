# Snowflake Docs

A Cortex Code skill that answers Snowflake product questions by searching the official documentation via the [Cortex Knowledge Extension (CKE)](https://app.snowflake.com/marketplace/listing/GZSTZ67BY9OQ4) Cortex Search service.

## How it works

The skill uses a [Cortex Search Service](https://docs.snowflake.com/en/user-guide/snowflake-cortex/cortex-search/cortex-search-overview) — a fully managed, serverless search engine built into Snowflake that indexes and retrieves unstructured data via hybrid (vector + keyword) search.

1. On first use, the skill checks whether the CKE is installed in your account. If not, it installs it automatically from the Snowflake Marketplace.
2. For every question, it queries the CKE search service and returns a grounded answer with links to the relevant documentation pages.

## Why this approach over web search or web fetch

- **No URL hallucinations** — canonical source URLs are returned directly from the search index, not generated by the model
- **Fewer tokens** — a single SQL call retrieves relevant chunks; no multi-hop web fetching or HTML parsing required
- **Always up to date** — the CKE index stays in sync with the published Snowflake documentation

## Usage

The skill activates automatically for Snowflake documentation questions, or you can invoke it explicitly:

```
$snowflake-docs How do I create a dynamic table?
```

Trigger phrases include any question about Snowflake SQL syntax, features, best practices, configuration, or troubleshooting.
